Performance platform

The Performance Platform is for comparing the performance of UK Government services. It's intended for civil servants, but accessible to everyone.

Government services should meet user's needs in a cost-effective and efficient way. To make sure this happens, we collect standardised data to compare how different services are doing.

The data is available to download for free by anyone.

About the data

We collect data about transactions. A transaction forms part of an end-to-end service; It's the process which people go through to create, update, or delete their information or status with Government.

We collect two types of data about transactions: Characteristics (theme, type, etc.) and Facts (cost, usage, etc).

Use the Characteristics to make sure you're doing fair comparisons of the Facts.

Characteristics

Term Definition Example values
Channel We collect transaction data seperately for each delivery channel. Online, paper, or phone.
User need The need the service which the transaction is part of solves. As a _____, I need to ________, so that _________.
Delivery organisation Split by department and public body. Full list found here.
Theme The high level theme the transaction fits into. Driving, business, etc.
Type What type of interaction the user has. Register an item, Apply for financial support, etc.

Facts

Cost per transaction The amount spent by Government to deliver the transaction. This figure does not include capital expenditure (set-up costs) or money paid as part of the service (e.g benefits or grants).
Usage The number of individual transactions completed. Bulk transactions are counted seperately - for example, paying tax on 3 vehicles at the same time counts as 3 transactions.
