Transaction performance

Download data about the performance of transactions used to deliver Government services.

Citizens make use of Government services by interacting with one or more transactions through online or offline channels. A transaction is the whole process that a person needs to go through to create, update, or delete the information that Government holds about an individual, legal entity or physical item.

About the data

The data is provided as a CSV file. A line in the file represents one transaction and is made up of data items, each of which is either a characteristic, metric or some other supporting information about the transaction.

Characteristics are useful for sorting, filtering and aggregating data in the file, and for comparing metrics across transactions on a like-for-like basis.

Metrics indicate how many citizens used a transaction during a period of time, and the total operating cost (excluding fixed costs) of meeting this demand.

Supporting information is included about what a transaction does, where it can be found, and who is responsible for the data provided about it.

Here is a list of the data items that will be provided for each transaction when you choose to download all data:

Characteristics

Department
The department responsible for the transaction.
Agency/body
The agency or body managing the transaction, if any.
Category
Transactions are categorised by type and by service.
Frequency of interaction
How often the transaction is accessed by a citizen.
Transactions dependent on
Other transactions which must be completed before the transaction can be completed.
Key components
Key components used in the delivery of the transaction.
GaaP components
Government as a Platform components used in the delivery of the transaction, if any.
Manual activity
Indicates whether the transaction requires manual activity in the back office.

Metrics

Started transactions
How many times the transaction was started during a time period.
Completed transactions
How many times the transaction was completed successfully during a time period.
Total operating cost
Total operating cost of meeting the demand for the transaction during a time period (excluding fixed costs).
% department operating cost
Total operating cost of the transaction as a percentage of the total operating cost of its department during a time period (excluding fixed costs).

Supporting information

Description
Description of the transaction.
URL
URL of the transaction's start page or home page.
Contact for data
Name and email of person responsible for the data.
